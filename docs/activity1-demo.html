üéØ Key Security Concepts

1. SQL Injection Prevention

Parameterized Queries: Use parameters instead of string concatenation

// ‚ùå Vulnerable string sql = $"SELECT * FROM Users WHERE Id = {userId}"; // ‚úÖ Secure string sql = "SELECT * FROM Users WHERE Id = @userId"; command.Parameters.AddWithValue("@userId", userId);

2. Secure Password Storage

Password Hashing: Never store passwords in plain text

// ‚úÖ Secure password hashing with salt public static string HashPassword(string password, out string salt) { salt = Convert.ToBase64String(RandomNumberGenerator.GetBytes(32)); using var pbkdf2 = new Rfc2898DeriveBytes(password, Convert.FromBase64String(salt), 100000, HashAlgorithmName.SHA256); return Convert.ToBase64String(pbkdf2.GetBytes(32)); }

3. Input Validation

Validate Everything: Never trust user input

// ‚úÖ Comprehensive input validation public bool ValidateInput(string input, int maxLength, string pattern) { if (string.IsNullOrWhiteSpace(input)) return false; if (input.Length > maxLength) return false; if (!Regex.IsMatch(input, pattern)) return false; // Check for common SQL injection patterns var sqlPatterns = new[] { "'", "--", "/*", "*/", "xp_", "sp_" }; return !sqlPatterns.Any(pattern => input.ToLower().Contains(pattern)); }
